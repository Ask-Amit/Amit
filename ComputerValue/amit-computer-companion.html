The Troubleshooting Record

What Happened — Step by Step

After I picked up the machine, I started working with Amit — an AI companion — to troubleshoot stability problems. Everything below is the actual process we ran through. I'm sharing it with you because you built this machine and you deserve the full picture.

1
Symptom — Unexpected Shutdowns
The machine began powering off unexpectedly. Not blue screens — just shutting down. Repeated. Unpredictable. Happened during normal use, not just under heavy load.
2
PASSED
Phase 1 — Rule Out Software
Ran full diagnostic pass before touching hardware:

• sfc /scannow — Windows system files: no violations
• Windows hardware error log: 0 recorded errors
• Full virus scan: zero threats
• Windows Update service: was set to Manual — corrected to Automatic
• Drive health: all drives healthy

Software ruled out as cause.
3
PASSED
Phase 2 — Temperature and Power
Monitored via HWiNFO64 under load:

• CPU (Ryzen 9 3900X): 82.6°C peak — within acceptable range
• GPU (RTX 3070): 68°C peak — healthy
• Motherboard voltages: 12V / 5V / 3.3V — all reading normal

Thermal and power issues ruled out.
4
CAUSE FOUND
Phase 3 — RAM Isolation
With software, temperature, and power all clean — isolated to RAM. Removed sticks two at a time and tested stability in each configuration. Process took multiple sessions (~6 hours total).

Two specific sticks identified as the cause. Every time those sticks were in the machine, shutdowns returned. With them removed, machine ran stable.

Result: Two sticks pulled. Machine stable on two sticks in slots 2 and 4.
5
Replacement Sticks Received — Attempt 1
Installed the two replacement G.Skill F4-3600C18-8GVK sticks into slots 1 and 3 alongside the working sticks in slots 2 and 4.

Machine booted. BIOS prompt appeared: "Memory configuration changed." Pressed F2 (load defaults). Machine booted into Windows — all 32GB recognized and showing correctly.

Attempted to enter BIOS to enable XMP. Machine locked up completely. Frozen — no response. Hard power-off required.

Removed replacement sticks from slots 1 and 3. Machine booted normally on two sticks.
6
Between Attempts — Additional Steps Tried
• Full power-off, disconnected from wall — cleared residual charge
• Removed and reseated GPU to eliminate as variable
• Attempted shutdown /r /fw /t 0 to boot directly to BIOS firmware — did not trigger as expected
• Used Windows Settings → Recovery → UEFI Firmware Settings path successfully on two-stick config
• Consulted a second source suggestion (Ryzen Master) — evaluated, not applicable to this issue
7
FAILED — SAME PATTERN
Replacement Sticks — Attempt 2
Reseated all four sticks. Powered on.

Immediate: slow fan mode. Monitor stayed black. No POST. No BIOS prompt. No signal. Nothing.

Powered off. Removed sticks from slots 1 and 3. Machine came back immediately.
8
HYPOTHESIS
What the Pattern Suggests
The replacement sticks may not be defective. The pattern is consistent with a board-level issue in slots 1 and 3 specifically. Every time anything is installed in those slots — the machine fails. Slots 2 and 4 have never caused a problem.

Worth considering: The two sticks originally identified as "bad" were in slots 1 and 3. It's possible those sticks aren't defective — it's possible the slots caused them to fail. The behavior is identical either way.
