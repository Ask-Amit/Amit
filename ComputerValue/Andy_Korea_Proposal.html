The core transaction loop

🔍

Full Diagnostic Suite

RAM (slot isolation, pass/fail), drive health (SMART + surface scan), GPU (temp under load, VRAM integrity), CPU (thermal, core performance, voltage), network, OS integrity. Amit knows every possible output. Explains every result at the level the person in front of the screen can act on.

📊

Honest Market Valuation

A range, not a number. Built from component depreciation curves + sold prices (eBay closed listings — not asking prices). Health score positions the machine within the range. Methodology visible. Timestamped. Buyer reruns any time to get today's value.

🏷️

PCV-ID — Hardware-Locked Identity

PCV-2026-RYZ9-RTX3070-K240-X1TB-A7X3
Generated from CPU + GPU + drive + board + date + random salt. Cannot be transferred. Physical possession photo required — builder photographs machine showing unique generated code.

⭐

Builder + Buyer Reputation

Both parties carry scores. Builder: pass rate, buyer ratings, configuration miss rate, did-they-certify-before-delivery. Buyer: dispute history, did post-purchase scan match. Progression: Provisional → Certified Builder (10+ builds, 90%+ pass rate) → Master Certified Builder (50+).

🔐

Anti-Fraud Hardware Fingerprint

Builder certifies before delivery. Buyer certifies after receiving. App compares hardware fingerprints — same machine confirms honest transaction. Mismatch opens dispute with both parties' evidence already captured. No arbitrator needed. The system has the evidence.

💰

Escrow Payment (Stage 4)

Buyer commits funds before machine ships. Funds hold until fingerprint match confirmed. In-person or shipped — same mechanism. Builder can't be walked away from. Buyer can't be defrauded. Neither trusts the other. Both trust the system.

The Upgrade Advisor — "Before you spend that money"

When a user considers an upgrade, Computer Value analyzes their actual machine and gives one of three answers — before they spend anything. The answer comes in seconds, not MHz. A before/after timed task runs on their real machine so the difference is felt, not calculated.

✓ Yes — here's exactly why

RAM is the actual bottleneck. Adding more will produce a real, felt difference for what you do. Here's what it changes and by how much.

→ Not yet — here's what to do first

Your CPU is the ceiling. Spending $150 on RAM won't change how this machine feels. Here's what would — and the right time to come back.

✋ You won't feel it

Your machine already has headroom. Save your money. Come back in 60 days — Amit will tell you when the upgrade actually pays off.

Talking someone out of a bad upgrade is the highest-trust thing Computer Value can do. The person who was told "don't spend that $150" comes back when they're ready to buy the upgrade that actually matters — and trusts every recommendation after that.

Use-Case Fit Assessment — "Will this computer actually do what I need it to do?"

Most certification tools answer one question: is this machine healthy? Amit answers a different one: is this machine right for you?

At the start of a session, Amit asks the person to describe their intended environment — the programs they'll run, the kind of work they'll do, the network they'll operate in. Not specs. Plain language. "I'm doing video editing for a small studio." "I run three virtual machines at a time for dev work." "It's going on a network with 40 other machines running inventory software."

Amit maps those requirements to the hardware it just certified and returns a fit assessment — not a number alone, but a verdict with reasoning:

✓ This machine fits your workload

Based on your environment, this configuration has the headroom to do what you need. Here's where it performs well — and where it's closest to its ceiling.

→ It can do it — with one change

The core is solid. Your video work will run, but rendering will bottleneck on 16GB of RAM. Adding 16GB more puts you well inside the headroom this workload needs. Without it, you'll feel it.

✗ This machine isn't built for what you're describing

The GPU driver doesn't support the rendering pipeline your software requires. The network card isn't rated for your environment. Here's what this machine is suited for — and what to look for instead.

The output isn't abstract. Amit runs a timed benchmark task tailored to the stated workload — before any upgrade, and after — so the difference is felt, not calculated. "That operation currently takes 15 seconds on this machine. With the RAM upgrade, it takes 13 seconds." Not MHz. Not GB. Seconds. The kind of number a person can use to make a decision.

This changes who the tool serves. It's no longer just for buyers verifying a purchase — it's for anyone who needs to know whether a machine is going to work for a specific job before committing to it. A contractor buying a laptop for field use. A studio spec'ing out edit stations. A small business evaluating a server before onboarding it into their network.
